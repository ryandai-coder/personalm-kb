# M1：基建 + CLI 对话

> 状态：方案讨论中 ｜ 日期：2026-06-10

---

## 一、核心交互

就一件事：

```
> 今天面试字节三面挂了，心情很差

隐：听起来不太顺利。三面能走到这一步本身就不容易。
    已记到 §9 情绪 / §5 岗位。
```

输入一句话 → AI 回复 + 后台自动分类归档。没有命令、没有模式切换、没有复杂交互。就是一个对话循环。

---

## 二、后台自动做的事（用户无感）

```
用户输入
    │
    ▼
┌──────────┐
│  classify │  AI 判断分类标签（§9 情绪 + §5 岗位）
└────┬─────┘
     ▼
┌──────────┐
│write_note │  生成 YAML frontmatter → 写入 vault 对应目录
└────┬─────┘
     ▼
┌──────────┐
│  回执     │  简短附在回复末尾
└──────────┘
```

AI 通过 function calling 自动完成 classify + write_note，用户只需说话。

## 三、System Prompt 分层——M1 能做到什么程度

设计文档定义了四层注入：

```
┌────────────────────────────────┐
│ L0  硬规则                      │  M1 ✅ 能做
│     不评判、先共情、不模仿用户    │  纯文本规则，直接写进 prompt
│     引用时必须 [[路径]]          │
├────────────────────────────────┤
│ L1  Self Memory（用户画像）      │  M1 ❌ 做不了
│     价值观、核心矛盾、关键模式    │  需要冷启动蒸馏产出，M1 没数据
├────────────────────────────────┤
│ L2  RAG 上下文                  │  M1 ⚠️ 形式上有
│     从 vault 检索相关历史笔记    │  关键词搜索可用，但无语义质量
├────────────────────────────────┤
│ L3  行为翻译表                  │  M1 ❌ 做不了
│     "他说算了不一定真算了"       │  需要积累语料后蒸馏
└────────────────────────────────┘
```

**M1 实际注入内容**：L0 硬规则 + 20 类体系说明 + 当前人格描述。

L1/L2/L3 的注入点先在代码里留好（函数参数、占位符），但内容为空。后续每补一层，只改 prompt 不碰架构。

## 四、必要基建

| 基建 | 做什么 | 为什么必须 |
|------|--------|-----------|
| LLM 调用 | 调 DeepSeek API（function calling） | 大脑 |
| System Prompt | L0 硬规则 + 20 类说明 + 人格 | 行为规范 |
| Agent 编排 | tool use 循环（classify → write_note → 回执） | LLM 不会自己读写文件 |
| Vault 写入 | 根据分类确定目录，生成 frontmatter，写 Markdown | 核心产出 |
| Vault 读取 | 解析 Markdown + frontmatter | 以后搜索和引用需要 |
| 审计日志 | JSONL 记录每次写入的路径和 hash | 后悔药 |
| 配置 | .env 加载 API key + vault 路径 | 启动前提 |
| CLI | readline 循环，输入→输出 | 交互界面 |

## 五、M1 不要的

- 不要 `/think` `/search` `/persona` `/log` `/undo` 等命令 — 用户只管说话
- 不要 Khoj 语义搜索 — 关键词匹配足够
- 不要 Self Memory / 行为翻译表 — 没数据蒸馏不了
- 不要主动追问 — 复杂度留给 M2
- 不要语音 / Telegram / 多模型路由
